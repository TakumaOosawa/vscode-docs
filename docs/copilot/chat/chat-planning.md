# VS Code chatでの計画

Visual Studio Code chatのエージェントは、複雑なコーディングタスクを自律的に実行するのに役立ちます。planエージェントを使用すると、実装を開始する前に詳細な実装計画を作成でき、すべての要件が満たされていることを確認できます。todoリストを使用すると、エージェントは全体の目標に集中し続け、進捗を効果的に追跡できます。

計画とtodoを使用すると、コーディングを開始する前にエージェントと一緒に実装の詳細を構造化して確認でき、生成されるコードの品質と信頼性が向上します。計画とtodoは、より複雑で長いタスクを体系的に進めるためのより良いガイダンスもエージェントに提供します。

この記事では、planエージェントとtodoリストを含む、VS Code chatで開発タスクを調査して計画する方法について説明します。

## 詳細なタスク調査のためのplanエージェント

組み込みのplanエージェントは、実行する前に詳細な実装計画を作成するためにあなたと連携します。これにより、コード変更が行われる前にすべての要件が検討され、対処されることが保証されます。planエージェントは、計画があなたによってレビューされ承認されるまで、コード変更を一切行いません。承認後は、その計画を既定のエージェントに引き継ぐか、さらに改善、レビュー、またはチームでの議論のために計画を保存できます。

planエージェントは、次のことを目的として設計されています。

* 読み取り専用ツールとコードベース分析を使用してタスクを包括的に調査し、要件と制約を特定する
* タスクを管理しやすい実行可能なステップに分解し、あいまいな要件に対する未解決の質問を含める
* 標準化された計画形式に基づいて、ユーザーがレビューして反復できる簡潔な計画ドラフトを提示する

### タスクを計画する方法

1. `kb(workbench.action.chat.open)`を押してチャットビューを開き、エージェントのドロップダウンから**Plan**を選択します。

2. 大まかなタスク(機能、リファクタリング、バグなど)を入力して送信します。

    例:
    * "Implement a user authentication system with OAuth2 and JWT"
    * "Add unit tests for all API endpoints"
    * "Save doesn't reset the form fields after submission"

3. 提案された計画ドラフトをプレビューし、反復のためのフィードバックを提供します。

    planエージェントは、高レベルの概要とステップの内訳を提供し、明確化のための未解決の質問も含みます。

    ![チャットビューでplanエージェントによって生成されたサンプル計画を示すスクリーンショット。概要、ステップの内訳、未解決の質問が含まれます。](../images/chat-planning/plan-agent-sample.png)

    > [!TIP]
    > 実装の前に計画を改善するため、planモードのままにしてください。要件を明確にしたり、スコープを調整したり、未解決の質問に対処したりするために、複数回反復できます。これにより、コード変更が行われる前に確かな基盤が確保されます。

4. 最終化したら、ボタンを使用して計画の実装を開始するか、エディターで計画を開いてさらにレビューします。

    同じチャットセッションで計画を実装することも、バックグラウンドまたはクラウドの[エージェント セッション](/docs/copilot/agents/overview.md)を開始して、バックグラウンドで自律的に実装に取り組ませることもできます。

    計画の実装を開始するときでも、「UIから始めて」や「ステップ1と2だけ」などの補足指示を引き続き提供できます。

## todoリストで進捗を追跡する

複雑なタスクに取り組むとき、VS Codeのエージェントは進捗を追跡するためのtodoリストを作成します。todoリストは要求を個々のタスクに分解し、AIが各ステップを完了するにつれて自動的に更新されます。これにより進捗を監視しやすくなり、長時間実行されるタスクの間もエージェントが集中し続けられます。

<video src="../images/chat-planning/todo-list-demo.mp4" title="チャット要求の進捗を追跡するための、チャットビュー内のtodoリスト コントロールを示すビデオ。" autoplay loop controls muted></video>

> [!TIP]
> 「ステップ1を修正してxを行うようにする」や「別のタスクを追加する」などの自然言語を使用してtodoリストを更新できます。エージェントのtodoが想定どおりでない場合はリストをクリアできますが、それ以外の場合、エージェントはフィードバックと要求の作業中に収集する情報に基づいて更新を自動的に管理します。
